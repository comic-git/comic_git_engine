# Host repo root discovery and execution context

## Context

`comic_git_engine` is developed and used through a host `comic_git` repo, not as a standalone application.

Scripts may be launched manually by maintainers or end users from different starting locations. Requiring every caller to know the exact script-relative root path would add friction and make local usage more brittle.

## Decision

Allow scripts to discover the host repo root by walking upward until `your_content/` is found.

- The presence of `your_content/` is treated as the practical marker for the host repo root.
- Manual build and utility scripts should assume they are operating inside a host repo context.
- Developer documentation should still tell maintainers to run scripts from the host repo root, even though root discovery exists as a convenience and safeguard.

## Consequences

This makes local use more forgiving and better aligned with how real users invoke the system.

It also means:

- `your_content/` is part of the effective contract for identifying a valid host repo
- future refactors should not replace root discovery with a more rigid assumption unless the user workflow is preserved
- explicit repo-root parameters may still be worth adding later for testability, but they should complement this behavior rather than silently remove it
